# Wasabi 每日政策英语训练：自动任务说明书

> 本文件是 Wasabi AI Learning Project 的可迁移配置与长期维护文档。
> 最后更新：2026-06-24（欧路 OpenAPI 已配置）

## 1. 项目目标

### 长期目标

1. 能较轻松地阅读 *The Economist*。
2. 能阅读英文政策研究机构报告。
3. 能独立撰写英文政策分析和经济研究报告。

### 当前能力

- 英语水平约为 CEFR B1+ 至 B2。
- 能表达复杂观点，政策分析和职业认知能力较强。
- 主要瓶颈是政策词汇不足、语法准确度不高，以及缺少成熟的英文表达体系。

### 学习原则

- 优先使用真实、近期、可靠的英文材料。
- 尽量保留 *The Economist* 和英文智库文章的写作风格。
- 必要时轻度简化长句和复杂背景，但不改变原有观点。
- 学习单位以词组、句型和论证结构为主，不孤立背单词。
- 训练阅读理解，也训练摘要、评论和政策写作。
- 日常训练尽量项目化，并与实际工作领域相连接。

## 2. 自动任务时间

- 每天上午 **10:00** 运行。
- 时区：`Asia/Shanghai`。
- 周日的每日任务中额外加入周度英语进展测试。
- 自动任务名称：`Wasabi 每日政策英语训练与周测`。

## 3. 每日选材要求

材料难度应适合 B1+ 至 B2，并逐步接近 C1。

优先来源：

- *The Economist*
- NATIONAL GEOGRAPHIC
- bloomberg
- *Financial Times*
- MIT Technology Review
- Resolution Foundation
- Institute for Government
- Institute for Fiscal Studies
- OECD
- IMF
- UK National Audit Office
- 英国、美国政府和议会官方网站

材料应尽量围绕：

- 美国、欧洲经济和公共政策
- AI产业发展最新进展
- AI、数字政府和公共服务改革
- 住房、规划与区域发展
- 财政、生产率和生活水平
- 产业政策
- 人口和劳动力市场

若使用学习版文章，必须明确标注：

> 学习版，根据相关研究编写或轻度改写，并非原文转载。

## 4. 每日课程结构

正常每日训练控制在 30-45 分钟，包括：

1. 今日主题和简短中文导读。
2. 英文阅读材料及可靠来源链接。
3. 美式英语阅读音频。
4. 10 个政策或经济高频核心词组。
5. 3 个可迁移的政策写作句型。
6. 3 个阅读理解或论证结构问题。
7. 约 100-120 词的 `Reverse Summary`。
8. 根据艾宾浩斯节奏安排的旧课复习。
9. 我的回答区。
10. 批改与反馈区。
11. 明日重点。

### 美式阅读音频

每天应为当日英文阅读材料生成一段标准、自然、清晰的美式英语朗读音频，并插入 Notion 当日子页面。

要求：

- 音频内容使用当日课程中的学习版英文阅读材料，不朗读受版权限制的完整商业原文。
- 发音风格应接近自然的政策播客或新闻评论朗读，而不是生硬的机器声。
- 语速适合 B1+ 至 B2 学习者，清晰、流畅、略慢于正常母语播报。
- 遇到政策、经济、产业和学术关键词时应适度强调。
- 段落之间应有自然停顿。
- Notion 中必须清楚标注：

```text
说明：该音频为 AI 生成的美式英语朗读，用于听读训练，并非真人录音。
```

默认技术方案：

- 生成脚本：`work/generate-reading-audio.cjs`
- 输入文件：包含 `date`、`slug`、`title`、`text` 的 JSON 文件
- 本地输出目录：`outputs/audio`
- 文件名格式：`YYYY-MM-DD-slug.mp3`
- 公开访问地址：

```text
https://wasabilee714.github.io/wasabi-english/audio/YYYY-MM-DD-slug.mp3
```

默认使用 Google Cloud Text-to-Speech：

- 默认声音：`en-US-Chirp3-HD-Charon`
- 语言：`en-US`
- 格式：`MP3`
- 建议语速：`0.92`

OpenAI 文本转语音仅作为备用方案：

- 模型：`gpt-4o-mini-tts`
- 默认声音：`cedar`
- 可选备用声音：`marin`

生成命令示例：

```powershell
node work\generate-reading-audio.cjs work\audio-input-YYYY-MM-DD.json
```

生成后必须把音频发布到 GitHub Pages，使 Notion 中的音频链接能在电脑和手机打开：

```powershell
node work\publish-site-to-github.cjs "Add reading audio for YYYY-MM-DD"
```

若 `GOOGLE_TTS_API_KEY` 未配置、Google TTS 接口失败、备用 OpenAI 接口失败或发布失败，应在当前线程和 Notion 页面中准确写明：

```text
音频生成未完成：具体原因
```

但不得因此中断当天主课程、卡片生成、Notion 归档或欧路同步。

安全要求：

- `GOOGLE_TTS_API_KEY` 优先保存在本地环境变量中；若 Codex 环境变量不稳定，可保存在 `work/.secrets/google-tts-api-key.txt`。
- `work/.secrets/` 必须加入 `.gitignore`，仅作为本机私密配置目录。
- 不得写入 GitHub、Notion、Markdown 说明书、课程页面或聊天记录。
- 换电脑时需要重新配置该环境变量。

### 核心词组要求

- 优先选择完整搭配，而非孤立单词。
- 应提供中文含义和自然英文例句。
- 应适合经济、政策、学术或职业表达。

### 仿写句型要求

- 每个句型应表达一种可复用的逻辑功能。
- 仿写题尽量联系中国经济、产业、规划、城市治理或区域发展。
- 示例：
  - `X is essential for Y, but it is not sufficient on its own.`
  - `The main barrier is not X, but Y.`
  - `X may improve the short-term numbers even if it weakens Y in the future.`

## 5. 周度测试

每周日增加周测，衡量是否更接近真实目标，而非只给出考试分数。

周测包括：

1. 一段新的政策英语阅读材料。
2. 阅读理解和论证结构问题。
3. 本周核心词组主动回忆。
4. 100-150 词英文摘要。
5. 80-100 词短政策评论。
6. 对以下能力分别评价：
   - 阅读主旨
   - 论证理解
   - 政策词汇
   - 语法准确度
   - 写作结构
   - 独立分析
7. 给出下周最重要的 2-4 个训练重点。

## 7. 卡片小程序

### 文件

当前文件：

`outputs/wasabi-cards.html`

当前电脑的完整路径：

`D:\\codex\\02 Daily English Learning\\outputs\\wasabi-cards.html`

迁移时应复制整个项目，或至少复制：

- `WASABI_AUTOMATION_SPEC.md`
- `wasabi-cards.html`

### 每日写入

每天把以下内容追加到卡片程序的 `initialCards` 数组：

- 10 张课程核心词组卡
- 3 张课程仿写句型卡
- 当天筛选出的欧路生词卡

若已配置 Supabase 云端服务，每日写入本地卡片后还必须运行：

```powershell
node work\sync-cards-to-supabase.cjs
```

将完整且去重后的卡片目录上传到 `wasabi_cards`。若云端配置缺失或上传失败，必须准确报告，但不得影响本地课程和卡片生成。

每张卡必须包含：

```js
{
  id: "稳定且唯一的 ID",
  type: "phrase | pattern | eudic",
  front: "正面提示",
  back: "英文答案",
  example: "自然英文例句",
  challenge: "主动输出题"
}
```

课程卡 ID 格式：

- `YYYYMMDD-p01` 至 `YYYYMMDD-p10`
- `YYYYMMDD-s01` 至 `YYYYMMDD-s03`

欧路卡 ID 格式：

- `eudic-YYYYMMDD-标准化词形`

不得重复已有 ID 或相同英文答案。

### 卡片交互

- 一次只显示一张卡。
- 正面在中文提示下方显示一条英文文本输入框。
- 用户直接输入英文答案，再点击 `提交`；也可以按 Enter 提交。
- 答案页显示后，再按一次 Enter 进入下一张卡片。
- 正面不再显示 `忘记 / 模糊 / 记得` 三个按钮。
- 判定时忽略：
  - 英文字母大小写
  - 首尾空格和连续空格
  - 常见标点符号
- 核心英文内容必须与标准答案一致，才判定为答对。
- 提交后翻到背面，并：
  1. 明确显示 `答对了` 或 `答错了`；
  2. 显示用户输入的答案；
  3. 显示标准英文答案、例句和主动输出题；
  4. 自动播放标准答案的美式英语发音；
  5. 显示 `下一张` 按钮。
- 卡片高度必须随背面内容自动增长，答案、例句、主动输出题和 `下一张` 按钮不得越过卡片边框或互相遮挡。
- 切换下一张时停止上一段发音。
- 优先使用系统中的 `en-US` 自然语音；找不到时回退到其他英语语音。
- 当前建议语速：`0.88`。
- 判定结果和复习进度首先保存在浏览器 `localStorage` 中。

### 安卓 App 与跨设备同步

- 卡片程序采用 PWA 形式，可在安卓 Chrome 中安装到桌面并全屏运行。
- PWA 文件包括：
  - `manifest.webmanifest`
  - `sw.js`
  - `wasabi-icon.svg`
  - `wasabi-icon-192.png`
  - `wasabi-icon-512.png`
  - `wasabi-cloud-config.js`
- PWA 必须通过 HTTPS 发布；`file://` 仅保留为电脑本地备用入口，不能安装到安卓。
- Service Worker 缓存应用外壳，使已加载内容可以离线使用。
- 手机和电脑使用同一邮箱账号登录 Supabase，自动同步个人学习进度。
- 同步以单张卡片的 `updatedAt` 为合并依据，最近一次学习记录优先，不以整份本地数据粗暴覆盖云端。
- 每日新增卡片存入云端 `wasabi_cards` 表，App 登录后自动拉取。
- 用户个人进度存入 `wasabi_progress` 表，并由行级安全策略限制为本人访问。
- 复习队列按艾宾浩斯到期日、薄弱程度和逾期天数计算优先级；同一优先级内随机打散，避免每次都按原始卡片顺序出现。
- 默认每日复习目标为 20 张，可在 App 侧栏调整，范围为 5-80 张。
- 若到期卡超过目标，只抽取优先级最高的一批；若到期卡不足目标，则补入新卡或自由练习卡。
- 答错卡片仍会在本轮稍后再次出现，因此实际完成次数可能略高于每日目标。
- 每日目标设置会随云端学习进度同步。
- `supabaseAnonKey` 可以写入前端配置；`SUPABASE_SERVICE_ROLE_KEY` 只能保存在电脑环境变量中，绝不写入仓库、Notion 或聊天记录。
- 详细配置步骤见 `outputs/MOBILE_APP_SETUP.md`。

## 8. 间隔复习

默认复习节点：

- 新学当天：D0
- D+1
- D+3
- D+7
- D+14
- D+30

自动判定行为：

- `答错`：本轮稍后再次出现，间隔归零，并标记为薄弱卡。
- `答对`：依当前熟练度进入 D+3、D+7、D+14 或 D+30。

每日复习应优先安排：

1. 曾回答错误的卡片。
2. 最近答错或已标记为薄弱的卡片。
3. 当天到期卡片。
4. 当天新卡。

## 9. 欧路词典同步

当前本机脚本：

- 连接测试：`work/test-eudic-connection.cjs`

目标：每天读取欧路词典当天新加入生词库的英语词条，并加入学习卡片。

### 数据规则

- 使用欧路词典官方 OpenAPI，不从浏览器缓存中提取账号凭证。
- 按 `Asia/Shanghai` 时区筛选当天新增词。
- 不重复导入全部历史生词。
- 将单词标准化为小写后去重。
- 与已有卡片的 ID 和英文答案去重。
- 每天最多选择 10 个。
- 超过 10 个时，优先选择：
  - 与当天主题相关的词
  - 经济和政策词
  - 学术写作词
  - 用户查阅频率较高的词
- 若没有新增词，不虚构。
- 若 API 未授权或不可用，明确报告原因，但继续生成主课程。

### 欧路词卡结构

- 正面：精炼中文义或英文语境提示
- 背面：英文词及词性
- 例句：自然、适合政策或工作语境
- 补充：常见搭配或派生词
- 主动输出题：联系政策分析或用户工作
